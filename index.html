Cascata, Especifidade e Combinadores

Cascata o que é:

A regra da cascata no CSS é o que determina quais estilos serão aplicados a um elemento quando há múltiplas regras competindo entre si. É como um sistema de desempate que segue uma hierarquia bem definida.

Especificidade:

Especificidade é o sistema de pontuação que o CSS usa para decidir qual regra de estilo vence quando há várias competindo pelo mesmo elemento. Quanto mais “específico” o seletor, maior sua prioridade.

  • Tag: div, p, h1. 0,0,0,1: força relativa: Baixa.
  • Classe: .menu, .btn. 0,0,1,0: força relativa: Média.
  • ID: #header, #nav. 0,1,0,0: força relativa: Alta.

Quanto maior a pontuação, mais forte o seletor. A pontuação é lida como: (inline, ID, classe, tag).

Seletores:

  • Combinador de descendente: Seleciona qualquer h1 que esteja dentro de .primeiro, em qualquer nível de profundidade.
  • Combinador de filho direto: Seleciona apenas o h1 que é filho direto de .primeira.
  • Combinador de irmão adjacente: Seleciona o h1 que vem logo depois de outro h1, dentro de .primeira.

Resumo: pega o h1 que é irmão adjacente (vem logo depois) de outro h1.

Pseudo-Classes:

Existem algumas Pseudo-Classes, como: first-child, last-child, nth-child, nth-of-type.

  • :first-child: Seleciona o primeiro filho de um elemento pai, independente da tag.
  • :last-child: Seleciona o último filho de um elemento pai, também independente da tag.
  • :nth-child(n): Seleciona o n-ésimo filho do pai, independente da tag.
  • :nth-of-type(n): Seleciona o n-ésimo filho do tipo específico de elemento (ex.: o segundo

    entre vários filhos).

Existem algumas formulas que podemos usar no ":nth-child", como:

  • :nth-child(odd) → filhos em posições ímpares.
  • :nth-child(even) → filhos em posições pares.
  • :nth-child(3n) → cada terceiro filho (3, 6, 9…).

Diferenças entre Display Inline e Display Block:

O display: block possui as seguintes caracteristicas:

  • Ocupa toda a largura disponível do elemento pai.
  • Sempre começa em uma nova linha.
  • Pode ter largura (width), altura (height), margem, padding, etc.

Já o display: inline possui algumas coisas diferentes como:

  • Ocupa apenas o espaço necessário para seu conteúdo.
  • Não quebra linha automaticamente.
  • Não aceita width, height, margin-top, margin-bottom (mas aceita padding lateral).

Uma curiosidade é que existem propriedades que se aplicam no display: block que nao se aplicam ao display: inline como:

  • width / height.
  • margin-top/bottom.
  • padding (todos).
  • float / clear.

Existem também elementos são por padrão display inline e quais são display block, aqui estão alguns.

display: block: div, section, article , footer.

display: inline: span, strong, em, b, i, u.

Propriedades:

Todo elemento HTML é uma “caixa” composta por: margin, border, padding e o conteúdo. Vou explicar para você cada uma delas.

Margin:

Serve para criar espaço externo entre o elemento e os elementos ao redor. É possivel usar: margin-top, margin-right, margin-bottom, margin-leftmargin-inline-start, margin-inline-en(horizontal, respeita direção do texto)margin-block-start, margin-block-end (vertical, respeita escrita vertical ou horizontal).

Padding:

Serve para: criar espaço interno entre o conteúdo e a borda do elemento. É possivel usar: Pode usar: padding-top, padding-right, padding-bottom, padding-left, padding-inline-start, padding-inline-end, padding-block-start, padding-block-end.

Border:

Serve para: criar uma borda visível ao redor do elemento. É possivel usar: border-top, border-right, border-bottom, border-left, border-inline-start, border-inline-end, border-block-start, border-block-end. Cada lado pode ter: border-width, border-style, border-color

Shorthand:

Você pode usar 1 a 4 valores para margin, padding e border-width:

Sufixos:

  • top
  • bottom
  • left
  • right
  • inline-start
  • inline-end
  • block-start
  • block-end

Box-Sizing

Por padrão, o CSS usa o modelo de caixa content-box, onde:

  • width e height se referem apenas ao conteúdo.
  • padding e border são adicionados por fora, aumentando o tamanho total da caixa.Com box-sizing: border-box, o comportamento muda:
  • width e height incluem padding e border.
  • O tamanho total da caixa não aumenta.
